LuauDispatcher

Executes a Luau AST produced by LuauParser.

On average it is 10% faster than vLuau and 108x slower than the native roblox engine. You can see better benchmark results on the GitHub page

Example usage:

local Parser = require(path.to.LuauParser)
local Dispatcher = require(path.to.LuauDispatcher)

local parseOptions = {
	-- any options you want to pass to Parser.parse
	-- ex.
	allowDeclarationSyntax = true,
	noErrorLimit = true
}
local runtimeOptions = {
	-- any options you want to pass to Dispatcher.run
	-- if you do not pass in any options, the default options will be used
	-- defaults:
	allowEnvironmentModifications = true,
	rawGlobalEnabled = true, -- this might be the one you consider disabling
	overrideCertainGlobalsAtRuntime = true,
	instructionLimit = 10000000,
	noEnv = false,
	allowConsoleOutput = true,
}
local code = [[
	print("Hello, world!")
]]

local success, tree = Parser.parse(code, parseOptions)
local completion = Dispatcher.run(tree, runtimeOptions)
if completion.kind == "Normal" then
	print("Success!")
end

A benefit of LuauDispatcher is, in my opinion, it is a lot easier to interface with than something like vLuau. You can add new functions in LuauDispatcher.globals if you want the execution environment to have more methods to work with. Example: 

Dispatcher.globals.foo = function()
	print("foo")
end

local parseOptions = {
	...
}
local runtimeOptions = {
	...
}
local success, tree = Parser.parse("foo()", parseOptions)
Dispatcher.run(tree, runtimeOptions)

Scripts also get access to rawglobal(name, value) which will add a global to the environment that will persist across all scripts. This can cause issues if you are not careful, so it can be disabled in the runtime options.

Example:

local parseOptions = {
	...
}
local runtimeOptions = {
	...
}
local success, tree = Parser.parse('rawglobal("foo", function() print("Hello, world!") end)', parseOptions)
Dispatcher.run(tree, runtimeOptions)

success, tree = Parser.parse("foo()", parseOptions)
Dispatcher.run(tree, runtimeOptions)

Error messages are also heavily customized to be easily readable so you can track the issue better.

local success, tree = Parser.parse("print(1 + 1", parseOptions)
Dispatcher.run(tree, runtimeOptions)

NOTE

This library uses FastSignal for connections. If you see the type ScriptSignal in these docs, think of a it the same as an RbxScriptSignal.

Types​

Completion​

</>
interface Completion {
kind: "Normal" | "Return" | "Break" | "Continue"
values: {any}?--

only present if kind == "Return"

}

Describes how a statement or a whole LuauDispatcher.run call finished.

RuntimeOptions​

</>
interface RuntimeOptions {
rawGlobalEnabled: boolean?--

Allows scripts to use the rawglobal(name, value) function.

overrideCertainGlobalsAtRuntime: boolean?--

Overrides error, assert, warn, print, and various methods from game:GetService("TestService")

noEnv: boolean?--

Runs the script without libs and functions from Dispatcher.globals

instructionLimit: number?--

The maximum number of expressions/statements evaluated before stopping script execution (default 10,000,000).

allowEnvironmentModifications: boolean?--

Allows scripts to overwrite globals in the environment.

allowConsoleOutput: boolean?--

If true, output functions will be able to print to the Roblox console.

}

Properties​

RunningCount​

This item is read only and cannot be modified. Read Only
</>
LuauDispatcher.RunningCount: number

The number of ASTs currently executing inside LuauDispatcher.run.

RunningStatusChanged​

</>
LuauDispatcher.RunningStatusChanged: ScriptSignal<boolean>
Dispatcher.RunningStatusChanged:Connect(function(isRunning)
	print(isRunning and "started" or "stopped")
end)

Becomes true when running count reaches 0, and false when it goes above 0.

LuauRan​

</>
LuauDispatcher.LuauRan: ScriptSignal<boolean,{any}>

Fires after LuauDispatcher.run AST execution.

Dispatcher.LuauRan:Connect(function(success, result)
	if success then
		print("success")
	else
		print("failed")
	end
end)

result is the AST that was executed, or was attempted to be executed.

UnhandledReturn​

</>
LuauDispatcher.UnhandledReturn: ScriptSignal<...any>

Fires when a script returns at the top level (essentially returning a value to nothing)

Dispatcher.UnhandledReturn:Connect(function(...)
	print("returned", select("#", ...), "value(s):", ...)
end)
local options = { ... }
local success, tree = LuauParser.parse("return 1", options)
Dispatcher.run(tree)

-- Output: returned 1 value(s): 1

Errored​

</>
LuauDispatcher.Errored: ScriptSignal<any>

Fires whenever a script throws an error. It is recommended to disable allowConsoleOutput in RuntimeOptions when using this connection. The function that gets connected will receive the error message

Output​

</>
LuauDispatcher.Output: ScriptSignal<...any>

Fires whenever a script calls print. The function that gets connected will receive any argument passed to the print function 

Dispatcher.Output:Connect(function(...)
	print(...) -- This code would actually be redundant, but it demonstrates the functionality
end)

Warned​

</>
LuauDispatcher.Warned: ScriptSignal<...any>

Like LuauDispatcher.Output, but for warn.

LuauError​

</>
LuauDispatcher.LuauError: LuauError

The error-formatting module the Dispatcher uses. You can call Dispatcher.LuauError.isErrors() on an AST to check if it has any issues.

globals​

</>
LuauDispatcher.globals: {[string]: any}

Add your own globals here (before running) to give them to scripts:

Dispatcher.globals.MyApi = MyApi

You can also replace any globals, if you want to do that for some reason. This will not work if trying to replace a console-printing function, such as warn, print, error, etc. You should instead connect to LuauDispatcher.Output, LuauDispatcher.Warned, or LuauDispatcher.Errored.

A script can also call the rawglobal(name, value) function to set a global that will stay in this table. This can be disabled when running Dispatcher.run() by passing in an options table.

Functions​

run​

</>
LuauDispatcher.run(
tree: any,--

a parsed tree from LuauParser

options: RuntimeOptions?,
...: any--

Varargs passed to the script

) → Completion

Executes a tree produced by LuauParser.

Returns a Completion describing how the script finished: kind is "Normal" for a script that just ran, or "Return" if the script returned. Alternatively you can specifically monitor for returns by connecting to LuauDispatcher.UnhandledReturn.

Optionally, you can pass in an options table of type RuntimeOptions to enable or disable specific parser features. Default options: 

local runtimeOptions = {
	allowEnvironmentModifications = true,
	rawGlobalEnabled = true, -- this might be the one you consider disabling
	overrideCertainGlobalsAtRuntime = true,
	instructionLimit = 10000000,
	noEnv = false,
	allowConsoleOutput = true,
}

You may also pass in arguments to the script by passing them in after the options table. Ex. 

local success, tree = LuauParser.parse([[
	local msg1, msg2 = ...
	print(msg1.." "..msg2)
]])
local completion = Dispatcher.run(tree, {}, "Hello", "World")

evaluateNode​

</>
LuauDispatcher.evaluateNode(
node: AstNode,--

any single AST node

scope: Scope?,--

an optional scope to evaluate within

options: RuntimeOptions?
) → Completion | ...any

Evaluates a single node.

CAUTION

Prefer LuauDispatcher.run whenever possible. Unlike LuauDispatcher.run, this does not automatically check the tree for parse errors, wrap execution in a pcall, update LuauDispatcher.RunningCount, or fire RunningStatusChanged, LuauRan, or UnhandledReturn
